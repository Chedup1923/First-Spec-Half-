SELISE Stationery Portal
Key Features Required

1.	Authentication System
Welcome Page: Role selection (Admin/Employee) before login 
Login Flow:
Email verification with OTP (6digit code)
Password authentication
Remember me option
Register Page: Create new accounts with First name, last name, email, password confirmation
Department selection (for employees only)
Password strength indicator
Password match validation
Forgot Password: Multistep password reset with OTP verification
Microsoft Login: Direct button that requires registered email + password verification
Session Management: Remember login, session persistence

2.	Employee Dashboard
Overview Page:
Statistics cards (total requests, pending, approved, rejected)
Recent requests table
Quick access buttons

My Requests Page:
Table with search, sort, and filter options
Status badges (pending, approved, rejected)
Pagination (10 pages)
Request details on click

New Request Form:
Category selection dropdown
Item name input
Quantity input
Urgency level (low/medium/high)
Additional notes textarea
Submit/Clear buttons

Notifications Page:
View all notifications
Request status updates
Timestamps
Profile Page: Display user information
Settings Page: Theme toggle, accent color picker, data management



3.	Admin Dashboard
Overview Page:
Statistics (total requests, pending, approved, employees)
Pending requests table
Quick overview

All Requests Page:
Full request table with all details
Search by item/employee name
Status filter dropdown
Sort options (date, urgency)
Pagination
Action buttons (approve/reject)

Request Review Modal:
Show request details
Add admin notes
Approve or reject buttons

Employees Page:
List of all registered employees
Email, department, join date
Request count per employee
Profile & Settings: Same as employee

4.	Design & UI Requirements
Color Scheme: Dark/Light theme toggle
Dark: Deep navy background (080b14), light text
Accent colors: Indigo (default), Violet, Cyan, Rose, Emerald
Glassmorphism: backdrop blur effects
Typography:
Display font: 'Syne' (bold, headings)
Body font: 'DM Sans' (regular, readable)

Components:
Glassmorphic cards with blur
Smooth button interactions
Toast notifications (topright)
Modal dialogs with animations
Status badges with icons
Smooth page transitions
Animations: Use GSAP for smooth transitions and effects
Three.js Background: Interactive 3D particle background that responds to mouse movement



5.	Data Management
LocalStorage: Store all user data, requests, notifications
Demo Data: Prepopulated demo employees and sample requests
Request States: Pending -  Approved/Rejected
Notifications: Autogenerated when requests are reviewed
User Types: Two roles (Admin, Employee) with different access levels


6.	Features & Functionality
Search & Filter: Find requests by item name, employee name, category, status
Sorting: By date, urgency, status
Pagination: 10 items per page with navigation
Realtime Updates: Tables refresh immediately after actions
Form Validation: Email format, password strength, required fields
Breadcrumbs: Navigation path on each page
Sidebar Navigation: Collapsible menu with rolespecific items
Topbar: Title, notification bell, theme toggle, admin badge
Responsive Design: Mobilefriendly layout

7.	Admin Actions
Approve Requests: Set status to approved, send notification to employee
Reject Requests: Set status to rejected, add rejection reason as note
View Details: Modal showing full request information
Manage Employees: View employee list with statistics

8.	Notifications
Toast Messages: Success, error, warning, info types
Notification Panel: Rightside dropdown with all notifications
Notification Badge: Shows unread notification count
Autodismiss: Toasts disappear after 3 seconds
